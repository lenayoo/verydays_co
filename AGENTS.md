# Verydays Website Agent Instructions

You are responsible for maintaining and improving the Verydays website.

The goal is to make the website feel like a clean, product-focused, iOS-like brand page that clearly explains each app and leads users to download.

---

## Core Goal
- Make each page clear, simple, and useful
- Improve conversion (users click App Store / Google Play)
- Keep design minimal and consistent with the app

---

## Content Writing

### Page Description Rules
- Do NOT write generic text like:
  "This page is about..."
- Write user-focused descriptions:
  - What the app does
  - Why it is useful
  - When users should use it

### Writing Style
- Simple English
- Short sentences
- Friendly and calm tone
- Avoid technical jargon
- Make it feel like an iOS product page

### Example Structure

For each app page:

1. Title
2. One-line value
3. Short description (2~3 lines)
4. Key benefits (bullet points)

Example:

WaterDays

Stay hydrated, simply.

Track your daily water intake with a clean and minimal interface.
Check your progress instantly with widgets.

- Simple daily tracking
- Lock Screen & Home widgets
- Gentle and distraction-free design

---

## App Links (Important)

For each app page:

- Always include:
  - App Store link (iOS)
  - Google Play link (if available)

### Rules
- Place links clearly (top or bottom of page)
- Use consistent button labels:
  - "Download on the App Store"
  - "Get it on Google Play"

### If link is missing:
- Leave placeholder and mark clearly:
  - "Coming soon"

### Agent behavior
- Detect which app the page is about
- Automatically match the correct store links
- Ensure links are visible and easy to click

---

## Page Structure

Each page should follow:

1. Hero section
   - App name
   - Short tagline
   - Main screenshot or visual

2. Description section
3. Feature highlights
4. Download section (CTA)

---

## Design Direction

- iOS-like
- Clean and minimal
- Soft colors (especially soft blue)
- Spacious layout
- Not too childish
- Not too complex

---

## UI Guidelines

- Make text easy to read
- Use clear hierarchy:
  - Title > subtitle > body
- Keep enough spacing between sections
- Avoid clutter
- Use large touch-friendly buttons

---

## Consistency

- Keep all pages visually consistent
- Use same tone and structure across apps
- Maintain brand identity (Verydays)

---

## Improvements

- Always improve clarity
- Remove unnecessary text
- Simplify layout where possible
- Focus on what helps users understand quickly

---

## Do NOT

- Do not over-design
- Do not add complex animations
- Do not add unnecessary sections
- Do not write long paragraphs

---

## Priority

1. Clear explanation of the app
2. Easy access to download links
3. Clean and minimal design

---

## Working Style

- Prefer simple over perfect
- Make fast improvements
- Iterate quickly
- Focus on real user experience
